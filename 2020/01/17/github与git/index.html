前言

git与github是什么？有什么关系？

别看这两个很像，其实两个是完全不同的东西，Git与Github的关系相当于你的汽车和库房的关系。Git是你的车，Github是库房，那么代码就相当于库房里的木头。你伐的木(写的代码)通过装到车（git）才能运到你的库房。这样说就很容易理解了吧！

  • github: 存放代码的远程仓库。
  • git： 管理github的工具。

Git🛠

基本介绍

  • 分布式版本控制工具
工作流程
  1. 从远程仓库中克隆Git资源作为本地仓库
  2. 从本地仓库中checkout代码然后惊醒代码修改
  3. 在提交代码之前将代码提交到暂存区
  4. 提交修改 -> 提交到本地仓库 ，本地仓库中修改的个历史版本
  5. 修改完成之后，需要和团队共享代码时，可以push到远程仓库

前进后退

查看日志

操作：

  • 索引(推荐)：前进后退git reset --hard [索引]
  • ^ : 后退git reset --hard HEAD^
  • ~：后退git reset --hard HEAD~n

reset命令的三个参数：

  • --soft: 本地库移动HEAD指针

  • --mixed：本地库移动HEAD指针,重置暂存区

  • --hard:本地库移动HEAD指针,重置暂存区,重置工作区

找回永久删除的文件:

使用git reset --hard [索引值]可以找回删除的文件

git分支(⭐⭐)

相当于复制几份，然后同时开发。各个分支互不影响。

好处：

  • 并行开发多个分支，提高效率

  • 相互独立，一个分支开发失败，不会影响其他分支。

  1. 查看所有分支
  1. 创建分支

  1. 切换分支

  1. 合并：

    • 先切换到接受的分支上
    • 执行merge命令

  1. 解决冲突

    • 编辑文件删除特殊符号
    • 修改文件修改到满意程度，保存退出
    • git add [文件名] 或 git add .
    • git commit -m "日志" 注意：不带文件名

冲突？两个分支的同一个地方都修改了就会产生冲突？

Github🎪

基本概念

  1. 仓库 （Repository）

    • 存放你项目的地方，每个项目都存放一个仓库中

  2. 收藏(Star)

    • 收藏项目的人数

  3. 复制和克隆(Fork)

    • 复制别人的仓库到我的账号下创建一个新仓库
    • 独立存在的

  4. 发起请求(Pull Request)

    • 基于fork的，你在别人的项目上做了改进、行把他合并这时候可以发请求
    • 发起人觉得不错、这是可以合并

  5. 关注(Watch)

    • 关注项目，当项目更新时可以接收到通知

  6. 事物卡片(Issue)

    • 发现代码BUG,但是目前没有成型代码，需要讨论时用

注册

  1. 注册

  2. 验证邮箱(收不到邮箱上设置白名单)

提交基本流程

  1. 初始化本地库
  2. github上创建一个仓库
  3. 与远程库连接git remote add [github仓库地址]
    • 查看git remote -v
  4. 推送git push origin [分支名]

克隆基本流程

  1. 克隆：远程库下载到本地，创建origin远程地址别名，初始化本地库
    • git clone [仓库地址]
pull

代码将从远程拉去到本地并与本地的分支进行合并，所以pull = fatch + merge

fatch

  • 抓取远程库git fatch origin [分支名]

  • 次不会改变本地工作区的文件

  • 切换到从远程下载的分支git checkou origin/master

merge

本地仓库与远程仓库的合并

  • 站在本地master分支上git merge origin/master
